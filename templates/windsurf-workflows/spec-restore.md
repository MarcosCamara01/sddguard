---
description: Restore a spec folder from a snapshot in .sdd/snapshots/
---

Execute the /spec-restore command defined in .sdd/workflow.md.

The spec name is whatever you typed after the command (e.g. `/spec-restore auth-refresh`).

List available timestamps for the feature, confirm the target with the user, then copy the snapshot files over specs/<spec>/1-requirements.md, 2-plan.md, 3-tasks.md. Report what was restored and what was lost.
